Architecture Spec

ExpenseTracker Architecture

A detailed engineering breakdown of the mobile and backend integration layers.

01 / The Architectural Strategy

ExpenseTracker bridges native mobile hardware listening capabilities with direct cloud database logging. This design eliminates manual logs entirely by intercepting SMS debit confirmations, launching an overlay card, and running backend AI classifications on user notes.

The system operates in two core environments:

  • Android App Layer: Written in Kotlin/Java (native drivers) and TypeScript (React Native thread).
  • REST API Layer: Written in Node.js, Express, and MongoDB, integrating Google Gemini APIs for intelligent categorizations.

02 / Mobile Frontend Stack

The mobile application utilizes React Native CLI as its framework runner combined with native Android modules:

  • State Management: Redux Toolkit for unified reactive dashboard components.
  • Local Preferences Store: Native Android SharedPreferences, written dynamically on auth state changes.
  • UI Layer: Hardware overlays generated directly by the window manager system framework.

03 / Server Backend Stack

The backend acts as the single source of truth for user logs and reports:

  • API Framework: Node.js and Express with strict TypeScript schemas validated via Zod.
  • Database: MongoDB via Mongoose object modeling.
  • AI Categorizer: Google Gemini API integration resolving categories on demand.

01 / Native SMS Interception Flow

The background logging engine is designed to run even if the React Native app is closed, suspended, or excluded from memory. This is achieved by moving the entire interception path to native thread runtimes.

1. OS Interception (SmsReceiver.kt)

Android OS triggers the broadcast receiver statically defined in the manifest. The receiver extracts the body text, validates that it is a debit alert, and pulls out the currency value using regex.

2. Draw System Overlay (FloatingBubbleService.kt)

Launches a floating circular bubble. The bubble outline uses hardware-level ViewOutlineProvider clipping to wrap the application's round launcher icon dynamically inside a transparent circular ring.

3. Card UI Note Capture

Tapping the bubble opens a full card panel. The user types their context note (e.g., "for coffee to sujeet") and clicks Done.

4. Background HTTP Sync

The service reads saved user auth token keys from SharedPreferences and sends an explicit network request directly to the backend database server, closing the widget immediately after.

02 / Native Background HTTP Client

This Kotlin implementation executes the backend POST request directly from a background thread when the user saves their note:

private fun createExpenseNatively(context: Context, amountStr: String, noteText: String) {
    Thread {
        try {
            val sharedPref = context.getSharedPreferences("ExpenseTrackerPrefs", Context.MODE_PRIVATE)
            val token = sharedPref.getString("auth_token", null)
            val apiUrl = sharedPref.getString("api_url", null)
            
            val url = java.net.URL("$apiUrl/expenses")
            val conn = url.openConnection() as java.net.HttpURLConnection
            conn.requestMethod = "POST"
            conn.setRequestProperty("Content-Type", "application/json")
            conn.setRequestProperty("Authorization", "Bearer $token")
            conn.doOutput = true

            val jsonBody = org.json.JSONObject().apply {
                put("amount", amountStr.toDouble())
                put("note", noteText)
            }
            
            val postDataBytes = jsonBody.toString().toByteArray(Charsets.UTF_8)
            conn.outputStream.use { os ->
                os.write(postDataBytes)
                os.flush()
            }
            conn.disconnect()
        } catch (e: Exception) {
            android.util.Log.e("FloatingBubbleService", "Sync error: ${e.message}")
        }
    }.start()
}

01 / JWT Verification Middleware

Protects routing tables by intercepting incoming requests, extracting bearer tokens, verifying credentials, and populating session parameters:

export const authenticateToken = (req: AuthRequest, res: Response, next: NextFunction) => {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) return res.status(401).json({ message: 'No token provided' });

  jwt.verify(token, JWT_SECRET, (err, decoded: any) => {
    if (err) return res.status(403).json({ message: 'Failed to authenticate token' });
    req.userId = decoded.userId;
    next();
  });
};

02 / Creation & AI Categorization Route

Accepts raw notes on creation. If a note is provided, it calls Gemini to deduce categories and clean summaries before saving to MongoDB:

router.post('/', async (req: AuthRequest, res: Response) => {
  try {
    const validated = createExpenseSchema.parse(req.body);
    const { amount, category, description, note } = validated;

    let finalCategory = category || 'Other';
    let finalDescription = description || 'SMS Expense';

    if (note) {
      const aiResult = await categorizeExpenseWithAI(amount, finalDescription, note);
      finalCategory = aiResult.category;
      finalDescription = aiResult.description;
    }

    const expense = new Expense({
      userId: req.userId,
      amount,
      category: finalCategory,
      description: finalDescription,
      date: new Date()
    });

    await expense.save();
    return res.status(201).json({ success: true, expense });
  } catch (error) {
    return res.status(500).json({ message: 'Error creating expense' });
  }
});
